## A causa-raiz (L-C)
`extractText` devolve **um único `text` concatenado**; o parser **re-regexa esse blob**. Quando a análise roda, **a fonte de cada dado já se perdeu** → não dá para dizer "isto veio de JSON-LD do produto" vs "isto é prosa". Por isso (1) proveniência na UI (P5) segue impossível, (2) o risco-vizinho é combatido por heurística de prosa em vez de **escopo na origem**, e (3) consertamos sintoma a sintoma. P2.4 conserta a **forma**, não mais um sintoma.

## A mudança (incremental, NÃO big-bang)
O extrator passa a emitir, além do `text` (fallback), **candidatos estruturados por campo**:

```ts
type FieldSource = "structured" | "meta" | "visible-text" | "reader";
type FieldScope  = "product" | "page" | "catalog";
interface Candidate<T> { value: T; source: FieldSource; scope: FieldScope; raw: string; }
// ExtractResult ganha (opcional, aditivo):
candidates?: {
  fiber?: Candidate<string>[];
  gsm?: Candidate<number>[];
  weave?: Candidate<Weave>[];
  // imagem já tem identidade/escopo (G1); pode entrar aqui depois
};
```
- **Fontes** já existem no código (JSON-LD `Product` = `structured`+`product`; `<meta>` = `meta`+`page`; containers visíveis = `visible-text`+`page`; reader markdown = `reader`+`page`). P2.4 **rotula** o que já coletamos, em vez de jogar no blob.
- **`scope` vem do A3/L-B:** candidato de nó escopado ao produto (JSON-LD Product, JSON island casado por handle/SKU) = `product`; resto = `page`/`catalog`. **Escopo fecha o risco-vizinho na origem** (não por rejeição-de-prosa).
- **Parser consome candidatos** por **precedência**: `structured+product` → `meta` → `visible-text` → `reader`; cai no **blob (fallback)** quando não há candidato. Carrega a fonte no contrato: `findings.<campo>.source`.

## Migração campo a campo (de-risca)
Um campo por vez, **mantendo o blob como fallback** e o **corpus congelado** como rede:
1. **fiber/composição** primeiro (maior valor, mais fontes estruturadas).
2. **gsm**.
3. **weave**.
4. (imagem já normalizada no G1 — integrar por último, se valer.)

A cada campo: adicionar a emissão de candidatos na fonte → fazer o parser preferir o candidato → **re-baseline do corpus auditado à mão** (diff campo a campo) → só então o próximo. Se um campo regredir o corpus, para e revê.

## Consome / habilita
- **P5 (UI):** passa a rotular por proveniência real (`findings.*.source`) — "lido da página (JSON-LD)" × "inferido". Hoje P5 está bloqueado por **isto**.
- **#P2-B / L-B:** com `scope` no candidato, minerar `__NUXT__` vira seguro (só aceita `scope:product`). Destrava o bucket adiado **sem** heurística frágil.
- **L-A (b):** enriquecimento de read parcial por campo é trivial quando os fatos são candidatos mescláveis.

## Reproduzir antes / rede de segurança (A-workflow)
- **Corpus congelado** (`corpus.test.ts.snap`) é a rede: nenhum veredito do corpus muda sem auditoria manual do diff.
- Fixtures novos por campo: (a) mesma composição em JSON-LD **e** prosa conflitante → precedência escolhe `structured+product`; (b) JSON-LD de **vizinho** (`scope:catalog`) **não** vence o produto; (c) sem candidato → cai no blob, resultado **inalterado**.
- Teste de contrato: `findings.fiber.source` presente e correto.

## Riscos e mitigações
- **Regressão silenciosa do parser** (maior risco): migração 1-campo-por-vez + corpus congelado + auditoria de diff + blob como fallback. **Reversível por campo.**
- **Vizinho via candidato catalog:** precedência exige `scope:product` no topo; candidatos `scope:catalog` são **descartados** (decisão #P2.4-B confirmada) — aceitá-los, mesmo com confiança rebaixada, reabriria exatamente o risco-vizinho que o A3 existe para fechar.
- **Complexidade:** manter `text` + candidatos é dualidade temporária; documentar que o blob sai quando todos os campos-chave migrarem.

## Testes / validação
- Unit: emissão de candidatos por fonte/escopo; precedência; fallback-blob; contrato `findings.*.source`.
- Corpus re-baseline auditado por campo.
- `pnpm test`/`lint`/`build`; curl ao vivo (Asket/Norse: `source` = structured; Superdry: reader); **auditoria independente** antes de enviar (fase arriscada, guia de workflow).
